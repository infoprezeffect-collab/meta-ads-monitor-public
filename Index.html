Étape 1

Créer ton application Meta Developer

Tu vas créer une application qui te permet d'accéder aux données de tes campagnes via l'API de Meta.

1.1 — Créer l'application

  1. Va sur developers.facebook.com
  2. Connecte-toi avec le compte Facebook lié à ton Meta Business
  3. Clique sur "Mes applications" → "Créer une application"
  4. Sélectionne le type : "Entreprise"
  5. Nomme l'app Ads Monitor (pas de "Meta" dans le nom)
  6. Sélectionne ton Business Manager → "Créer"
⚠️ Note ton App ID et App Secret : Paramètres → Général → "Afficher"

1.2 — Ajouter les permissions Marketing API

  1. Menu gauche → "Ajouter un produit" → "Marketing API" → Configurer
  2. Dans Marketing API → "Autorisations"
  3. Coche ces 4 permissions : ads_read ads_management business_management read_insights

1.3 — Générer ton token long durée (60 jours)

  1. Va sur Graph API Explorer
  2. Important : sélectionne ton application Ads Monitor en haut à droite
  3. Clique "Générer un token" → coche ads_read, read_insights, business_management
  4. Accepte → copie le token court

Convertis-le en token 60 jours avec cette commande :

Windows :

curl.exe "https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=TON_APP_ID&client_secret=TON_APP_SECRET&fb_exchange_token=TON_TOKEN_COURT"

Mac/Linux :

curl "https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=TON_APP_ID&client_secret=TON_APP_SECRET&fb_exchange_token=TON_TOKEN_COURT"
✅ Tu reçois un access_token long → note-le, c'est ta clé principale

1.4 — Récupérer ton Ad Account ID

curl.exe "https://graph.facebook.com/v21.0/me/adaccounts?fields=id,name&access_token=TON_TOKEN_LONG"

Note l'ID qui commence par act_XXXXXXXXXX

Étape 2

Configurer Gmail pour l'envoi automatique

Tu vas créer un mot de passe d'application Gmail — un code spécial qui permet à ton script d'envoyer des emails en ton nom sans exposer ton vrai mot de passe.

2.1 — Activer la validation en 2 étapes

  1. Va sur myaccount.google.com/security
  2. Active "Validation en 2 étapes" si pas déjà fait

2.2 — Créer le mot de passe d'application

  1. Va sur myaccount.google.com/apppasswords
  2. Dans le champ "Nom", tape : Meta Ads Monitor
  3. Clique "Créer"
  4. Google génère un code de 16 caractères ex: abcd efgh ijkl mnop
  5. Enlève les espaces → abcdefghijklmnop
  6. Note ce code — il ne s'affiche qu'une seule fois !
⚠️ C'est ce code (pas ton mot de passe Gmail) que tu mettras dans la variable EMAIL_PASSWORD
✅ C'est tout pour Gmail ! Simple et sécurisé.
Étape 3

Obtenir ta clé API Anthropic (Claude)

Claude est l'IA qui va analyser tes données Meta et rédiger le rapport. Tu as besoin d'une clé API et d'un minimum de crédits.

Créer ton compte et ta clé API

  1. Va sur console.anthropic.com
  2. Crée ton compte ou connecte-toi
  3. Menu gauche → "API Keys"
  4. Clique "Create Key" → nomme-la Meta Ads Monitor
  5. Copie la clé — elle commence par sk-ant-api03-...
  6. Va dans "Billing" → ajoute minimum 5$ de crédits
💡 Coût estimé : environ 0,03$ à 0,05$ par rapport généré.
Pour des rapports quotidiens du lundi au vendredi → ~1$/mois seulement.

Vérifier que ça fonctionne

Tu peux tester ta clé rapidement :

curl https://api.anthropic.com/v1/messages \
  -H "x-api-key: TA_CLE_API" \
  -H "anthropic-version: 2023-06-01" \
  -H "content-type: application/json" \
  -d '{"model":"claude-opus-4-6","max_tokens":10,"messages":[{"role":"user","content":"test"}]}'
✅ Si tu vois un objet JSON avec "type":"message", ta clé est valide !
Étape 4

Installer Python et configurer les fichiers

4.1 — Installer Python

  1. Va sur python.org/downloads
  2. Télécharge Python 3.11 ou plus récent
  3. Crucial : lors de l'installation, coche ✅ "Add Python to PATH"
  4. Vérifie l'installation : ouvre PowerShell et tape python --version

4.2 — Télécharger les fichiers

⬇️ Télécharger le package complet (ZIP)

Dézipe dans un dossier dédié, par exemple :

Windows : C:\Users\TON_NOM\meta-ads-monitor\
Mac     : /Users/TON_NOM/meta-ads-monitor/

4.3 — Créer le fichier .env

Copie env.example et renomme-le .env. Ouvre-le et remplis avec tes vraies valeurs :

META_ACCESS_TOKEN=ton_token_long_duree_60_jours
META_APP_ID=1234567890
META_APP_SECRET=ta_cle_secrete
META_AD_ACCOUNT_ID=act_1234567890
ANTHROPIC_API_KEY=sk-ant-api03-xxxx
EMAIL_SENDER=ton@gmail.com
EMAIL_PASSWORD=abcdefghijklmnop
EMAIL_RECIPIENT=ton@gmail.com
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
⚠️ Ne partage JAMAIS le fichier .env — il contient tous tes mots de passe et clés API.

Étape 6

Déployer sur Railway

Railway est un serveur cloud qui lance ton script automatiquement chaque matin — même PC éteint, même en vacances.

6.1 — Créer un repo GitHub privé

  1. Va sur github.com/new
  2. Nom : meta-ads-monitor — Visibilité : Private
  3. Coche "Add README" → "Create repository"
  4. Upload ces fichiers uniquement : meta_ads_monitor.py requirements.txt Procfile railway.json
⚠️ Ne jamais uploader le fichier .env sur GitHub !

6.2 — Créer le projet Railway

  1. Va sur railway.app → "Login with GitHub"
  2. Clique "+ New" → "GitHub Repository"
  3. Sélectionne meta-ads-monitor

6.3 — Ajouter les variables d'environnement

Dans Railway → bloc meta-ads-monitor → onglet "Variables" → ajoute chaque ligne de ton .env :

META_ACCESS_TOKEN     → ton token 60j
META_APP_ID           → ton app id
META_APP_SECRET       → ton secret
META_AD_ACCOUNT_ID    → act_xxxxx
ANTHROPIC_API_KEY     → sk-ant-...
EMAIL_SENDER          → ton@gmail.com
EMAIL_PASSWORD        → mot de passe app 16 car.
EMAIL_RECIPIENT       → destinataire@gmail.com
SMTP_HOST             → smtp.gmail.com
SMTP_PORT             → 587

6.4 — Vérifier le déploiement

Dans l'onglet "Deployments", tu dois voir :

  • ✅ Deployment successful
  • ✅ Ready
  • 📅 Next in X hours
🚀 Ton assistant tourne désormais 24h/24 dans le cloud !
Étape 7

Comprendre ton rapport quotidien

📌 Important : l'assistant analyse uniquement la santé publicitaire — la qualité de la diffusion dans Meta. Il n'a pas accès aux ventes, inscriptions ou données post-clic. Ces données vivent dans ton CRM, hors de portée de Meta.
