Étape 1 sur 8

Créer ton application Meta Developer

🎓 Pourquoi cette étape ?

Meta ne laisse pas n'importe qui accéder aux données de campagnes par programme. Tu dois créer une "application" dans leur portail développeur — c'est en quelque sorte une carte d'identité pour ton script. Elle te donnera un token d'accès : une clé secrète qui autorise ton script à lire tes données publicitaires.

1.1 — Créer l'application

  1. Va sur developers.facebook.com
  2. Connecte-toi avec le compte Facebook lié à ton Meta Business
  3. Clique sur "Mes applications" → "Créer une application"
  4. Type : "Entreprise"
  5. Nom : Ads Monitor (Meta interdit "Meta" dans le nom)
  6. Sélectionne ton Business Manager → "Créer"
⚠️ Note ton App ID et ton App Secret quelque part — tu en auras besoin à l'étape suivante. (Paramètres → Général → "Afficher")

1.2 — Ajouter les permissions Marketing API

  1. Menu gauche → "Ajouter un produit" → cherche "Marketing API" → Configurer
  2. Dans Marketing API → onglet "Autorisations"
  3. Coche ces 4 permissions :
    ads_read ads_management business_management read_insights

1.3 — Générer ton token long durée (60 jours)

Le token est la "clé" que le script utilisera pour accéder à tes données. Il faut d'abord générer un token court, puis l'échanger contre un token valable 60 jours.

  1. Va sur Graph API Explorer
  2. Important : en haut à droite, sélectionne ton application Ads Monitor
  3. Clique "Générer un token d'accès" → coche ads_read, read_insights, business_management
  4. Accepte → copie le token court généré

Convertis-le en token 60 jours :

Windows :

curl.exe "https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=TON_APP_ID&client_secret=TON_APP_SECRET&fb_exchange_token=TON_TOKEN_COURT"

Mac / Linux :

curl "https://graph.facebook.com/v21.0/oauth/access_token?grant_type=fb_exchange_token&client_id=TON_APP_ID&client_secret=TON_APP_SECRET&fb_exchange_token=TON_TOKEN_COURT"
✅ Tu reçois un access_token très long — c'est ton token 60 jours. Note-le précieusement.

1.4 — Récupérer ton Ad Account ID

curl.exe "https://graph.facebook.com/v21.0/me/adaccounts?fields=id,name&access_token=TON_TOKEN_LONG"

Note l'ID qui commence par act_XXXXXXXXXX — c'est l'identifiant de ton compte publicitaire.

Étape 2 sur 8

Configurer Gmail pour l'envoi automatique

🎓 Pourquoi cette étape ?

Ton script a besoin d'envoyer des emails en ton nom. Pour des raisons de sécurité, Gmail ne permet pas à un programme d'utiliser ton mot de passe principal. Tu vas donc créer un "mot de passe d'application" — une clé spéciale, utilisable uniquement pour cette tâche, qui ne donne pas accès à tout ton compte.

2.1 — Activer la validation en 2 étapes

Prérequis obligatoire pour créer un mot de passe d'application.

  1. Va sur myaccount.google.com/security
  2. Active "Validation en 2 étapes" si pas déjà fait

2.2 — Créer le mot de passe d'application

  1. Va sur myaccount.google.com/apppasswords
  2. Dans le champ "Nom", tape Meta Ads Monitor
  3. Clique "Créer"
  4. Google génère un code de 16 caractères : ex. abcd efgh ijkl mnop
  5. Enlève les espaces → abcdefghijklmnop
  6. Note ce code — il ne s'affiche qu'une seule fois !
⚠️ C'est ce code (pas ton mot de passe Gmail habituel) que tu mettras dans la variable EMAIL_PASSWORD à l'étape 5.
✅ C'est tout pour Gmail. Simple, sécurisé, 5 minutes chrono.
Étape 3 sur 8

Obtenir ta clé API Anthropic (Claude)

🎓 Pourquoi cette étape ?

Claude (l'IA d'Anthropic) est le "cerveau" de ton assistant. C'est lui qui va lire les données brutes de Meta et les transformer en rapport clair et actionnable — comme le ferait un media buyer senior. Pour l'utiliser par programme, tu as besoin d'une clé API et d'un peu de crédits prépayés.

Créer ton compte et ta clé

  1. Va sur console.anthropic.com
  2. Crée ton compte ou connecte-toi
  3. Menu gauche → "API Keys"
  4. Clique "Create Key" → nomme-la Meta Ads Monitor
  5. Copie la clé — elle commence par sk-ant-api03-...
  6. Va dans "Billing" → ajoute minimum 5$ de crédits
💰 Coût réel : environ 0,03$ à 0,05$ par rapport généré.
Pour des rapports quotidiens + hebdomadaires → moins de 1,50$/mois.

Comment est calculé le coût ?

Claude facture au nombre de "tokens" traités (un token ≈ ¾ de mot). Chaque rapport envoie les données de tes campagnes à Claude et reçoit en retour le HTML du rapport. Le tout représente environ 3000 à 5000 tokens par rapport — d'où le coût très faible.

✅ Avec 5$ de crédits, tu as de quoi générer plus de 100 rapports — soit plusieurs mois d'utilisation.

Installer Python et configurer les fichiers

🎓 Pourquoi cette étape ?

Python est le langage de programmation dans lequel le script est écrit. Tu n'as pas besoin de savoir coder — tu vas juste l'installer et remplir un fichier de configuration avec tes clés. C'est l'équivalent de brancher une box internet : tu n'as pas besoin de comprendre comment ça marche, juste de connecter les bons câbles.

5.1 — Installer Python

  1. Va sur python.org/downloads
  2. Télécharge Python 3.11 ou plus récent
  3. Crucial ! Lors de l'installation, coche ✅ "Add Python to PATH" — sans ça, rien ne fonctionnera
  4. Vérifie en ouvrant PowerShell et en tapant python --version

5.2 — Télécharger les fichiers

⬇️ Télécharger le package complet (ZIP)

Dézipe le contenu dans un dossier dédié :

Windows : C:\Users\TON_NOM\meta-ads-monitor\
Mac     : /Users/TON_NOM/meta-ads-monitor/

5.3 — Créer le fichier de configuration .env

Le fichier .env est ton fichier de configuration secret. Il contient toutes tes clés API et identifiants. Copie env.example, renomme-le .env, et remplis chaque ligne avec tes vraies valeurs :

META_ACCESS_TOKEN=ton_token_long_duree_60_jours
META_APP_ID=1234567890
META_APP_SECRET=ta_cle_secrete
META_AD_ACCOUNT_ID=act_1234567890
ANTHROPIC_API_KEY=sk-ant-api03-xxxx
EMAIL_SENDER=ton@gmail.com
EMAIL_PASSWORD=abcdefghijklmnop
EMAIL_RECIPIENT=ton@gmail.com
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
⚠️ Ne partage JAMAIS ce fichier .env — ni en email, ni sur GitHub. Il contient toutes tes clés d'accès.

Étape 7 sur 8

Comprendre ton rapport quotidien

📌 Important à comprendre : l'assistant analyse uniquement la santé publicitaire — la qualité de la diffusion dans Meta. Il n'a pas accès à tes ventes, inscriptions ou données post-clic. Ces données vivent dans ton CRM ou ton tunnel, hors de portée de Meta. L'absence de ventes dans le rapport n'est donc jamais un signal d'alarme.

Pourquoi on structure en TOF / MOF / BOF ?

C'est la logique du tunnel publicitaire. Chaque niveau cible des personnes à un stade différent de leur relation avec toi — et nécessite une approche et des créas différentes.

TOF
Top of Funnel — Audience froide
Des personnes qui ne te connaissent pas encore. Objectif : capter l'attention, créer de la notoriété. KPI prioritaire : CPM et CTR. Attention : si tu n'alimentes pas ton TOF, ton audience s'épuise vite.
MOF
Middle of Funnel — Audience tiède
Des personnes qui ont déjà interagi (vu une vidéo, visité ton site, liké un post). Objectif : nourrir l'intérêt, renforcer la confiance. KPI prioritaire : CPC et fréquence.
BOF
Bottom of Funnel — Audience chaude
Des personnes proches de convertir — visiteurs récents, abandons de panier, liste email. Objectif : déclencher le lead ou l'achat. KPI prioritaire : CPL et volume de leads.
💡 L'équilibre TOF/MOF/BOF est clé. Si tu mets tout ton budget en BOF sans alimenter le TOF, ton audience chaude s'épuise en quelques semaines et ton CPL explose. L'assistant surveille cet équilibre et t'alerte si un niveau est sous-financé.

❓ FAQ

Le rapport indique 0 leads — c'est un problème ? ▼
Non. Le script récupère uniquement les conversions trackées par le pixel Meta. Si ton tunnel (quiz, mini-cours, optin) n'est pas relié au pixel, Meta ne voit pas les leads. Surveille le CPL et le CTR — si ces indicateurs sont bons, ta diffusion est saine. Les leads réels sont dans ton CRM.
Le rapport commente mes ventes — est-ce normal ? ▼
Non, et si c'est le cas c'est une erreur de configuration du prompt. L'assistant est paramétré pour analyser uniquement la santé publicitaire, pas le business. Il ne doit jamais conclure que "le tunnel est cassé" ou que "la monétisation ne fonctionne pas" — ces données sont hors de sa portée.
Le rapport arrive à quelle heure ? ▼
À 9h00 UTC du lundi au vendredi, soit 10h00 heure française en été (CEST) et 10h00 en hiver (CET). Le rapport analyse la journée d'hier (journée complète) — pas la journée en cours qui n'a que quelques minutes de données à 9h00.
Que faire quand le token Meta expire ? ▼
Le token expire après 60 jours. Lance le script refresh_token.py inclus dans le package — il génère automatiquement un nouveau token. Mets ensuite à jour la variable META_ACCESS_TOKEN dans les Variables de ton projet Railway. Mets un rappel dans ton agenda 55 jours après chaque renouvellement.
Puis-je surveiller plusieurs comptes publicitaires ? ▼
Pas en configuration de base — le script est conçu pour un seul META_AD_ACCOUNT_ID. Il est techniquement possible de le modifier pour boucler sur plusieurs comptes, mais cela nécessite des ajustements dans le code. Contacte-nous si tu as ce besoin.
Mes données sont-elles sécurisées ? ▼
Oui. Le script tourne sur ton propre compte Railway avec tes propres clés API. Ni le code, ni les données ne transitent par des serveurs tiers. Tes clés sont stockées dans les variables d'environnement de Railway, chiffrées au repos. Le repo GitHub est privé — personne d'autre ne peut y accéder.
Que se passe-t-il si je n'ai pas de campagnes actives un jour ? ▼
Le script tourne quand même mais le rapport sera vide ou très court, indiquant simplement l'absence de dépenses et de données. Aucune erreur ne sera générée — c'est le comportement attendu.
Puis-je changer l'heure d'envoi ? ▼
Oui. Dans le fichier railway.json, la ligne "cronSchedule": "0 9 * * 1-5" définit l'heure. Le format est "minute heure jour mois jour_semaine". 0 8 * * 1-5 = 8h00 UTC (9h00 heure française). Modifie ce fichier sur GitHub et Railway redéploiera automatiquement.
Combien coûte vraiment ce système chaque mois ? ▼
Environ 6 à 7€/mois tout compris : ~5$/mois Railway (plan Hobby) + ~1$/mois de tokens Claude pour les rapports quotidiens et hebdomadaires. Tout le reste (Meta API, GitHub, Gmail) est gratuit.

Modifier le script — guide précis

🎓 Avant de modifier quoi que ce soit

Ouvre meta_ads_monitor.py avec VSCode (gratuit, recommandé) ou Notepad++. Fais toujours une copie de sauvegarde avant de modifier — renomme l'original en meta_ads_monitor_backup.py au cas où.

🔴 Modifier les seuils d'alerte

Les seuils déclenchent les alertes dans le rapport. Par défaut ils sont calibrés pour la lead gen généraliste — adapte-les à ta réalité.

Seuil CPL (Coût par Lead)

Cherche la ligne ~340 dans le script :

# AVANT (défaut)
  CPL (Coût par Lead) :
    Une hausse de +20% sur 3 jours consécutifs = signal d'alerte

Et dans le prompt IA, cherche ~362 :

# AVANT
🔴 CPL > 15€

# APRÈS — exemple si ton CPL cible est 8€
🔴 CPL > 8€
💡 Remplace 15 par ton CPL cible réel. Si tu ne sais pas encore quel est ton CPL cible, garde 15€ par défaut.

Seuil CTR

Cherche ~363 :

# AVANT (défaut)
🔴 CTR < 1%

# APRÈS selon ton secteur
🔴 CTR < 0.8%   ← B2B (normal d'avoir un CTR plus bas)
🔴 CTR < 1.5%   ← E-commerce (audience plus réactive)

Seuil Fréquence

Cherche ~364 :

# AVANT (défaut)
🔴 Fréquence > 3

# APRÈS si ton audience est petite (< 50 000 personnes)
🔴 Fréquence > 2   ← seuil plus bas car saturation plus rapide

🕘 Modifier l'heure d'envoi

L'heure est configurée dans railway.json, pas dans le script Python.

Ouvre railway.json sur GitHub et cherche :

# AVANT (9h00 UTC = 10h00 heure française)
"cronSchedule": "0 9 * * 1-5"

# APRÈS — exemples
"cronSchedule": "0 7 * * 1-5"   ← 8h00 heure française (été)
"cronSchedule": "0 6 * * 1-5"   ← 7h00 heure française (été)
"cronSchedule": "30 7 * * 1-5"  ← 8h30 heure française (été)
⚠️ L'heure dans Railway est en UTC. En été (CEST) : UTC+2, donc -2h. En hiver (CET) : UTC+1, donc -1h.

🤖 Modifier le modèle IA utilisé

Par défaut le script utilise claude-opus-4-6 — le plus puissant. Si tu veux réduire les coûts, tu peux passer à un modèle moins cher.

Cherche ~ligne 417 (et ~ligne 722 pour le rapport hebdomadaire) :

# AVANT (plus puissant, plus cher)
"model": "claude-opus-4-6",

# APRÈS (plus rapide, moins cher ~5x)
"model": "claude-haiku-4-5-20251001",
⚠️ Le modèle Haiku est beaucoup moins cher mais les rapports seront moins détaillés et les recommandations moins nuancées. Recommandé uniquement si tu as un budget très serré.

📊 Modifier la fenêtre d'analyse (7 jours → autre)

Par défaut les comparaisons se font sur 7 jours. Cherche ~ligne 230 dans la fonction collect_all_data() :

# AVANT (fenêtre 7 jours)
week_range = get_date_range(8, 2)   # J-8 à J-2

# APRÈS — fenêtre 14 jours (si tu as des cycles plus longs)
week_range = get_date_range(15, 2)  # J-15 à J-2

📊 Sur le rapport

Le rapport indique 0 leads — c'est un problème ? ▼
Non. Le script récupère uniquement les conversions trackées par le pixel Meta. Si ton tunnel (quiz, mini-cours, optin) n'est pas relié au pixel, Meta ne voit pas les leads. Surveille le CPL et le CTR — si ces indicateurs sont bons, ta diffusion est saine. Les leads réels sont dans ton CRM.
Le rapport dit que "le tunnel est cassé" — que faire ? ▼
C'est une erreur de configuration du prompt. L'assistant est paramétré pour ne jamais conclure sur le tunnel business. Si tu vois ce type de commentaire, vérifie que tu as bien le fichier meta_ads_monitor.py version 2.0 (celui fourni dans ce guide). Télécharge à nouveau le package et remplace le fichier.
Pourquoi le rapport analyse "hier" et pas "aujourd'hui" ? ▼
Le rapport est envoyé à 9h00 — à cette heure, "aujourd'hui" n'a que quelques minutes de données, ce qui fausserait complètement l'analyse. Le script analyse donc hier (journée complète) et compare à avant-hier + la moyenne des 7 derniers jours. C'est la logique professionnelle d'un media buyer.
Je n'ai reçu aucun rapport ce matin — que vérifier ? ▼
Dans l'ordre : 1) Vérifie les spams Gmail. 2) Va sur Railway → onglet Deployments → clique sur le dernier déploiement → "Deploy Logs" pour voir l'erreur. 3) Vérifie que le token Meta n'est pas expiré (60 jours). 4) Vérifie le solde Anthropic sur console.anthropic.com/settings/billing.

🔧 Sur la configuration

Que faire quand le token Meta expire (tous les 60 jours) ? ▼
1) Lance python refresh_token.py dans ton dossier — il génère automatiquement un nouveau token et l'affiche. 2) Sur Railway → Variables → mets à jour META_ACCESS_TOKEN avec le nouveau token. 3) Railway redéploie automatiquement. ✅ Mets un rappel Google Agenda 55 jours après chaque renouvellement.
Puis-je surveiller plusieurs comptes Meta Ads ? ▼
Pas en configuration de base — le script gère un seul META_AD_ACCOUNT_ID. Il est possible de le modifier pour gérer plusieurs comptes (un rapport par compte ou un rapport consolidé), mais cela nécessite des ajustements dans le code. Contacte-nous si tu as ce besoin spécifique.
Comment changer l'adresse email de réception ? ▼
Sur Railway → Variables → modifie la valeur de EMAIL_RECIPIENT. Tu peux mettre n'importe quelle adresse email. Railway redéploie automatiquement et le prochain rapport sera envoyé à la nouvelle adresse.
Mes données sont-elles sécurisées ? ▼
Oui. Le script tourne sur ton propre compte Railway avec tes propres clés API — personne d'autre n'y a accès. Tes clés sont stockées dans les variables d'environnement Railway, chiffrées au repos. Ton repo GitHub est privé. Aucune donnée ne transite par des serveurs Prez'effect.

💰 Sur les coûts

Combien coûte vraiment ce système chaque mois ? ▼
Railway : ~5$/mois (plan Hobby). Claude IA : ~0.03$ à 0.05$ par rapport × ~22 rapports/mois = ~0.70 à 1.10$/mois. Total : environ 6 à 7€/mois. Tout le reste (Meta API, GitHub, Gmail) est gratuit.
Que se passe-t-il si je manque de crédits Anthropic ? ▼
Le script plante avec une erreur 401 Anthropic et tu ne reçois pas de rapport ce jour-là. Aucune donnée n'est perdue. Il suffit de recharger des crédits sur console.anthropic.com et le lendemain le rapport reprend normalement.

🚀 Sur les performances

Que faire si mes campagnes ne sont pas structurées en TOF/MOF/BOF ? ▼
Pas de problème — la détection TOF/MOF/BOF est automatique et basée sur les mots-clés dans les noms de tes campagnes. Si tes campagnes ne contiennent pas ces mots-clés, elles seront toutes classées "Non classifié". L'analyse fonctionne quand même — tu perdras juste la partie "analyse par niveau". Pour en profiter, renomme tes campagnes avec TOF, MOF ou BOF dans le titre.
Ça fonctionne pour quel type de campagnes Meta ? ▼
Toutes les campagnes Facebook et Instagram : trafic, génération de leads, conversions, notoriété, engagement. Le script récupère les données de toutes les campagnes actives dans ton compte publicitaire.
Puis-je l'utiliser sans Railway — sur mon PC uniquement ? ▼
Oui — c'est même par là qu'on a commencé. Lance simplement python meta_ads_monitor.py manuellement quand tu veux un rapport. Pour l'automatiser sur PC, utilise le Planificateur de tâches Windows (voir étape 6 de la version locale dans le guide). L'inconvénient : ton PC doit être allumé et connecté à l'heure prévue.
